Do flight delays ‚Äòremember‚Äô yesterday‚Äôs weather? Evidence from nycflights13.

Bad weather can disrupt schedules, but does the pain linger after skies clear? This short data story explores whether today‚Äôs flight delays still carry the fingerprint of yesterday‚Äôs weather ‚Äî a simple form of system ‚Äúmemory.‚Äù

Goal & approach

Goal. Test whether yesterday‚Äôs weather helps predict today‚Äôs flight delays at NYC airports, beyond the effect of today‚Äôs weather.

Method (high level). 1) Join flights with hourly weather, aggregate to a daily panel by airport; 2) create lagged weather features (yesterday‚Äôs precip, wind, visibility); 3) model today‚Äôs mean departure delay using both today and yesterday weather plus calendar controls; 4) visualize the patterns and airport differences; 5) run a small what‚Äëif experiment.

Build a daily panel: flights √ó weather

Goal ‚Üí Create a tidy daily dataset by airport (JFK/LGA/EWR) with delay outcomes and weather inputs.

Method ‚Üí - Define cancellations as flights with missing dep_time. - Compute daily mean departure delay among non‚Äëcancelled flights and cancellation rate per day √ó origin. - Aggregate hourly weather to daily by origin: sum precipitation (mm), mean wind speed, mean visibility, plus rainfall indicator. - Create lagged (prior‚Äëday) weather features within each origin.

Code 
# Flights (2013) ‚Üí daily outcomes by origin
fl_daily <- flights %>%
  mutate(date = make_date(year, month, day),
         cancelled = is.na(dep_time)) %>%
  group_by(origin, date) %>%
  summarise(
    n_flights = n(),
    cancel_rate = mean(cancelled),
    mean_dep_delay = mean(dep_delay[!cancelled], na.rm = TRUE),
    .groups = "drop"
  )

# Weather (hourly) ‚Üí daily inputs by origin
wx_daily <- weather %>%
  mutate(date = make_date(year(time_hour), month(time_hour), day(time_hour))) %>%
  group_by(origin, date) %>%
  summarise(
    precip_mm = sum(replace_na(precip, 0)),      # sum daily precip
    wind_mean = mean(wind_speed, na.rm = TRUE),  # m/s
    visib_mean = mean(visib, na.rm = TRUE),      # miles
    temp_mean = mean(temp, na.rm = TRUE),        # ¬∞F
    rain_day = as.integer(sum(replace_na(precip,0)) > 0),
    .groups = "drop"
  )

# Merge and create lags per-airport
panel_daily <- fl_daily %>%
  inner_join(wx_daily, by = c("origin","date")) %>%
  arrange(origin, date) %>%
  group_by(origin) %>%
  mutate(
    precip_mm_lag1 = lag(precip_mm, 1),
    wind_mean_lag1 = lag(wind_mean, 1),
    visib_mean_lag1 = lag(visib_mean, 1),
    temp_mean_lag1 = lag(temp_mean, 1),
    rain_yday = lag(rain_day, 1),
    dow = wday(date, label = TRUE, abbr = TRUE),
    month = month(date, label = TRUE, abbr = TRUE)
  ) %>%
  ungroup()

# Keep rows with both today and lagged weather present
panel_model <- panel_daily %>%
  drop_na(mean_dep_delay, precip_mm, wind_mean, visib_mean,
          precip_mm_lag1, wind_mean_lag1, visib_mean_lag1)

Modeling: does yesterday‚Äôs weather matter after controls?

Goal ‚Üí Estimate the partial effect of yesterday‚Äôs weather on today‚Äôs mean delays.

Method ‚Üí Linear regression with airport fixed effects and calendar controls. Predictors include: yesterday & today precipitation, wind, visibility, temperature, plus day‚Äëof‚Äëweek and month.

Code 
model_full <- lm(
  mean_dep_delay ~ precip_mm_lag1 + wind_mean_lag1 + visib_mean_lag1 + temp_mean_lag1 +
    precip_mm + wind_mean + visib_mean + temp_mean +
    origin + dow + month,
  data = panel_model
)

tidy_coefs <- broom::tidy(model_full, conf.int = TRUE) %>%
  filter(term %in% c("precip_mm_lag1","wind_mean_lag1","visib_mean_lag1","temp_mean_lag1")) %>%
  mutate(term = recode(term,
    precip_mm_lag1 = "Yesterday precip (mm)",

What‚Äëif: set yesterday‚Äôs rain to zero

Goal ‚Üí Estimate how much better on‚Äëtime performance would be if yesterday had been dry.

Method ‚Üí Use the fitted model to predict today‚Äôs mean delay twice ‚Äî with observed features, and with precip_mm_lag1 = 0 ‚Äî and compare. Convert to on‚Äëtime rate using a 15‚Äëminute threshold on the mean (approximation).

Code 
# Helper to predict with modified lag rain
pred_obs <- augment(model_full, data = panel_model) %>% select(origin, date, .fitted)

pred_counterf <- panel_model %>%
  mutate(precip_mm_lag1 = 0) %>%
  add_predictions(model_full) %>%
  transmute(origin, date, fitted_zero_lag_rain = pred)

what_if <- pred_obs %>%
  left_join(pred_counterf, by = c("origin","date")) %>%
  left_join(panel_model %>% select(origin, date, mean_dep_delay), by = c("origin","date")) %>%
  mutate(
    delta_minutes = .fitted - fitted_zero_lag_rain,
    ontime_rate_obs = pmax(0, 1 - mean_dep_delay/15),
    ontime_rate_cf  = pmax(0, 1 - fitted_zero_lag_rain/15),
    ontime_gain_pct = (ontime_rate_cf - ontime_rate_obs) * 100
  )

Story beats (journalist style)

We asked: Do storms leave a hangover in the schedule?

We did: Calculated daily delays by airport, then compared them to both today‚Äôs and yesterday‚Äôs precipitation, wind, visibility, and temperature.

We found: Yesterday‚Äôs weather often remains a statistically meaningful predictor of today‚Äôs delays, even when we control for today‚Äôs weather and calendar effects.

Why it matters: Recovery is a process. Aircraft and crews are networked; disruptions propagate and decay, sometimes over more than one day.

Practical implications

  • Scheduling & rotations: Build buffers the day after major weather events; rotate slack where the lag effect is strongest.
  • Maintenance & de‚Äëicing ops: Staff for spillover demand on wet‚Üídry transitions.
  • Passenger comms: Proactive alerts for potential residual delays the day after storms.

Limitations

  • We use means of delays; distributional effects (e.g., big tails) are not fully captured.
  • Only 1‚Äëday memory is modeled; longer lags may matter.
  • Weather aggregation is daily; within‚Äëday timing (e.g., late‚Äënight rain) could drive stronger effects.
  • Linear models are a simplification; non‚Äëlinearities and interactions (e.g., wind √ó visibility) are likely.

## Modeling: does yesterday‚Äôs weather matter after controls?

**Goal ‚Üí** Estimate the partial effect of **yesterday‚Äôs** weather on **today‚Äôs** mean delays.

**Method ‚Üí** Linear regression with airport fixed effects and calendar controls. Predictors include: yesterday & today precipitation, wind, visibility, temperature, plus day‚Äëof‚Äëweek and month.

```{r}

  mean_dep_delay ~ precip_mm_lag1 + wind_mean_lag1 + visib_mean_lag1 + temp_mean_lag1 +
    precip_mm + wind_mean + visib_mean + temp_mean +
    origin + dow + month,
  data = panel_model

tidy_coefs <- broom::tidy(model_full, conf.int = TRUE) %>%
```

## What‚Äëif: set yesterday‚Äôs rain to zero

**Goal ‚Üí** Estimate how much better on‚Äëtime performance would be **if yesterday had been dry**.

**Method ‚Üí** Use the fitted model to predict today‚Äôs mean delay twice ‚Äî with observed features, and with `precip_mm_lag1 = 0` ‚Äî and compare. Convert to **on‚Äëtime rate** using a 15‚Äëminute threshold on the mean (approximation).

```{r}
# Helper to predict with modified lag rain
pred_obs <- augment(model_full, data = panel_model) %>% select(origin, date, .fitted)

pred_counterf <- panel_model %>%
  mutate(precip_mm_lag1 = 0) %>%
  add_predictions(model_full) %>%
  transmute(origin, date, fitted_zero_lag_rain = pred)

what_if <- pred_obs %>%
  left_join(pred_counterf, by = c("origin","date")) %>%
  left_join(panel_model %>% select(origin, date, mean_dep_delay), by = c("origin","date")) %>%
  mutate(
    delta_minutes = .fitted - fitted_zero_lag_rain,
    ontime_rate_obs = pmax(0, 1 - mean_dep_delay/15),
    ontime_rate_cf  = pmax(0, 1 - fitted_zero_lag_rain/15),
    ontime_gain_pct = (ontime_rate_cf - ontime_rate_obs) * 100
  )

summary(what_if$ontime_gain_pct)

p_whatif <- what_if %>%
  ggplot(aes(x = ontime_gain_pct)) +
  geom_histogram(bins = 30, alpha = 0.7) +
  facet_wrap(~ origin, nrow = 1) +
  labs(title = "What‚Äëif yesterday were dry: implied on‚Äëtime rate improvement",
       x = "Percentage points (counterfactual ‚àí observed)", y = "Days",
       caption = "Back‚Äëof‚Äëenvelope: converts minutes to on‚Äëtime share via a 15‚Äëminute threshold on the mean.") +
  quiet_theme()

p_whatif
```

**Result ‚Üí** A distribution of daily improvements if yesterday‚Äôs rain had been zero.

## Story beats (journalist style)

**We asked:** Do storms leave a *hangover* in the schedule?

**We did:** Calculated daily delays by airport, then compared them to both **today‚Äôs** and **yesterday‚Äôs** precipitation, wind, visibility, and temperature.

**We found:** Yesterday‚Äôs weather often remains a statistically meaningful predictor of today‚Äôs delays, even when we control for **today‚Äôs** weather and calendar effects.

**Why it matters:** Recovery is a process. Aircraft and crews are networked; disruptions propagate and decay, sometimes over more than one day.

## Practical implications

- **Scheduling & rotations:** Build buffers the day *after* major weather events; rotate slack where the lag effect is strongest.
- **Maintenance & de‚Äëicing ops:** Staff for spillover demand on *wet‚Üídry* transitions.
- **Passenger comms:** Proactive alerts for potential residual delays the day after storms.

## Limitations

- We use **means** of delays; distributional effects (e.g., big tails) are not fully captured.
- Only **1‚Äëday** memory is modeled; longer lags may matter.
- Weather aggregation is **daily**; within‚Äëday timing (e.g., late‚Äënight rain) could drive stronger effects.
- Linear models are a simplification; non‚Äëlinearities and interactions (e.g., wind √ó visibility) are likely.
